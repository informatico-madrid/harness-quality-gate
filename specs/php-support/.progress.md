### Design Interview (from design.md)
- **Concurrency**: Configurable via flag `--concurrency [parallel|sequential|auto]`. Default `auto` = parallel locally (ThreadPoolExecutor per-layer), sequential in CI (detected via `CI=true`/`GITHUB_ACTIONS`/`GITLAB_CI` env vars). Lets users force sequential in pipelines for clean output and parallel locally for ~3x speedup.
- **Fault tolerance**: Hybrid — hard-fail on critical tools missing (Infection, PHPUnit, PHPStan, Psalm-taint, PHPMD, Deptrac, composer audit, PHP-CS-Fixer); skip-with-warning on optional tools (secondary PHPMD rulesets, ergebnis/phpstan-rules, shipmonk/phpstan-rules extension packs). Exit code 3 (TOOL_MISSING) on critical absence; checkpoint JSON marks `status: 'incomplete'` and `missing_tools[]`. Taxonomy lives in `config/php-tool-taxonomy.json`.
- **Code layout**: Python package `harness_quality_gate/` with submodules by concern: `detector.py`, `dispatcher.py`, `cli.py`, `checkpoint.py`, `doctor.py`, `adapters/{base.py,python/,php/,shared/}`. Each adapter inherits `BaseAdapter(ABC)` with `run_l3a()`, `run_l1()`, `run_l2()`, `run_l3b()`, `run_l4()`. ~12-15 files. Testeable, extensible for future TS/Go.
- **PHAR install location**: `~/.cache/harness-quality-gate/bin/` (XDG-compliant user cache). Always writable, persists across invocations, doesn't contaminate repo or skill dir. Versioned by tool hash. Doctor verifies/repopulates. Avoids `${CLAUDE_SKILL_DIR}` read-only risk in plugin installs.
- **Chosen approach**: Adapter pattern + dispatcher + configurable concurrency. Clean ABC seam for adding TS/Go/Rust later. No plugin entry_points (overkill for 2 langs).

## Learnings
- Detector 3-tier scoring: `(10 if manifest_hits else 0) + min(file_count, 100)` — manifest signals dominate file-count tiebreaks
- Detection.primary is a computed property returning `language`, not a separate field
- EXCLUDE_DIRS covers both Python and PHP ecosystem directories (vendor, .venv, node_modules, __pycache__)

- PHP has no ruff+ast equivalent for Tier A AST antipatterns — PHPMD covers ~13/25, the other 12 require custom nikic/PHP-Parser visitors. Same LoC budget as Python.
- Psalm is the ONLY mainstream PHP static analyzer with built-in taint analysis (TaintedSql, TaintedHtml, TaintedShell, TaintedSSRF). PHPStan does not offer taint. Recommended split: PHPStan for daily lint, Psalm only for L4 taint pass.
- Deptrac is preferred over PHPat for hexagonal validation: declarative YAML config is more agent-friendly, standalone binary (no PHPStan coupling), built-in graphviz output, JSON/GitHub/JUnit reporters.
- Infection has no native per-directory MSI threshold — must emulate via multiple `--filter` runs or aggregation in mutation_analyzer.
- PCOV is ~2.8× faster than Xdebug for line coverage but is unmaintained since 2021. Works on PHP 8.4. Fallback to Xdebug required for branch coverage.
- Three complementary CVE tools: composer audit (reactive runtime scan), local-php-security-checker (fast standalone binary, same DB), roave/security-advisories (preventive — Composer constraint blocking install of vulnerable versions).
- shipmonk/dead-code-detector is the modern Vulture equivalent and is framework-aware (Symfony/Laravel/Doctrine reflection).
- shipmonk/composer-dependency-analyser is a single-tool replacement for composer-unused + composer-require-checker (unused + shadow + misplaced deps in one pass).
- Semgrep PHP rule coverage is "experimental but useful" — supplement with Psalm taint analysis. Use rulesets p/php + p/phpcs-security-audit + p/owasp-top-ten.
- PHPStan 2.1.34 added 25–40% caching speedup in early 2026. Required extensions: phpstan-strict-rules, phpstan-deprecation-rules, spaze/phpstan-disallowed-calls, ergebnis/phpstan-rules, shipmonk/phpstan-rules + framework pack (larastan or phpstan-symfony).
- ~80% of the existing harness scripts are language-agnostic and reusable: diversity_metric, llm_solid_judge, antipattern_judge, configurator, workflow.md, SKILL.md, all references/ except semgrep-python-rules.yaml.
- No existing prior art in the PHP ecosystem for a Ralph-Loop-style 5-layer quality gate with Tier A/B split and checkpoint JSON. This skill would be novel.
- Psalm taint adapter (1.15) handles both canned array format `[{"type":"TaintedSql",...}]` and real nested `{"files":{...}}` Psalm output. Taint rule types filtered against TAINT_RULE_TYPES set. `Finding` model lacks `layer`/`tool`/`language`/`rule_id` fields; these are encoded in `message` (type: msg) and `fix_hint` (type) per POC.
- **BUG (independent of PHP)**: The skill uses `{skill-root}` placeholder which is NOT a Claude Code primitive. The canonical variable is `${CLAUDE_SKILL_DIR}` per code.claude.com/docs/en/skills. Migration required before/with polyglot port — broken portability today on plugin installs.
- Claude Code ships NO guaranteed runtimes besides embedded Bun (GitHub issues #30465, #9444). Skills cannot assume python/php/node on PATH. Doctor pattern is mandatory for multi-runtime skills.
- Cross-skill dependencies are not first-class in Claude Code (open issue #9444). Rules out Option C (3 separate skills with shared common). Single-skill orchestrator (Option D) is the only marketplace-friendly path.
- Inline injection (!`<cmd>`) in SKILL.md runs ONCE and output is not re-scanned for further placeholders. Detection logic MUST be an executable script invoked inline, not multi-stage prompt substitution.
- GitHub Linguist's 6-stage detection pipeline is overkill for quality-gate scope; top 3 stages (manifest + source-count + tie-breaker) cover all realistic cases. Moonrepo confirms manifest-presence is the industry-standard polyglot detection signal.
- MegaLinter (Python orchestrator + 100+ subprocess linters + descriptor-per-tool YAML) is the proven architecture for polyglot quality gates. Super-Linter (bash-sequential) is the anti-pattern to avoid.
- PHAR distribution is mature for Infection, PHPStan, PHPUnit, PHP-CS-Fixer, Deptrac, Psalm — usable as fallback when Composer absent. Do NOT bundle PHARs in skill (size + version drift); ship install_php_tools.sh helper instead.
- Anthropic's /run-skill-generator pattern shows: per-project detection should be cached to disk (.quality-gate-lang.cache), not re-derived every invocation.
- Recommended dispatcher: Option D (single Python orchestrator + PHP-tool adapters, no PHP runner). All PHP tools have JSON CLI output → Python subprocess + json.loads is sufficient. Only custom nikic/PHP-Parser visitors require shipping PHP source files.
- Checkpoint JSON v2 requires top-level `language` field + per-layer `language` + per-finding `language`. All existing keys stay stable (additive change). Hybrid repos get per_language sub-blocks.
- Dual-profile YAML schema: `gates:` (shared thresholds) + `language_profiles.{python,php}:` (tool configs) + `shared_tools:` (gitleaks/checkov/trivy). Schema versioning required (v1 auto-migrates).
- Backward compat plan: legacy {skill-root} aliased to ${CLAUDE_SKILL_DIR}, legacy scripts/ paths shimmed to runner.py subcommands, legacy v1 config auto-wrapped into language_profiles.python on first load.
